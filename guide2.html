How To Create A Minecraft Server

Who Should Read This Guide?

From personal experience, hosting Minecraft servers from free online platforms were always a nuisance. Slower than desired, underperforming, barely any control, and allocated resources were very scarce. Instead, if you created a Cloud Compute Instance through OCI, you can get around the rabbit hole and create something far more powerful.

Prerequsite: Installing TMUX On Your Server

TMUX is a application that can be installed through your SSH. Its major use is that it can create "headless windows" that act as separate secure shell hosts. It is very useful because these windows persist after the closure of the secure shell (in this case, Putty) that connects with the server. Multiple of these windows can be run simultaneously, too. TMUX is what elevates a localhost only server to a dedicated 24/7 server that is playable anytime.

To install TMUX, go to the terminal of the SSH and execute this command:

sudo yum install tmux

Enter "y" after the prompt and TMUX should be installed onto your instance.

Convenient TMUX Commands

Action Command
Creating a new session 
tmux new -s [SESSION NAME]
Killing a session 
tmux kill-ses -t [SESSION NAME]
Attaching to a session 
tmux a -t [SESSION NAME]
Listing sessions 
tmux ls
Exiting a session 
Ctrl B + D
Renaming a session 
Ctrl B + $

NOTE: More information can be found here at https://tmuxcheatsheet.com/

Opening The Ports

Previously, you had create Ingress Rules through the Primary VNIC. Unfortunately, that is only half of the work in order to get your ports online. Before the server can start, they must be opened for you to connect to the server. By default, the port for Minecraft servers is 25565. Other ports may be used, however. To open a port, run these commands in the SSH terminal:

Action Command
Opening the TCP Port 
sudo firewall-cmd --permanent --zone=public --add-port=25565/tcp
Opening the UDP Port 
sudo firewall-cmd --permanent --zone=public --add-port=25565/udp
Reload/Update Ports 
sudo firewall-cmd --reload

Downloading Necessary Files

different Minecraft versions require different versions of JAVA. Recent versions of Minecraft require JAVA version 20, while older versions of Minecraft require JAVA version 8. If you have not downloaded JAVA, now is the time to do it. To install JAVA, first check which version you want to download by executing this command in the SSH terminal:

sudo yum list java\*

After finding which version you want to install, run this command next and hit "y" to confirm:

sudo yum install [JAVA VERSION YOU WANT FROM THE LIST]

NOTE: Exclude the keyword "sudo" if you are executing it in a TMUX window, or else it will not work!

After installing JAVA, download the Minecraft server JAR file from https://www.minecraft.net/en-us/download/server for the recent version of Minecraft, or browse throughout the internet for the right version. Transfer this file into your instance by using WinSCP.

If you want to install modpack servers, some providers include CurseForge, Feed The Beast, and Modrinth. The files already have everything bundled up and ready to go!

Running The Server

If you had not already done so, create a new TMUX window. Head to the directory that you're server is in and ensure the files have the correct permissions. Here is another convenient table to show possible commands to start the server:

Action Generic Command Example
Basic Start Command 
java -Xmx[MEMORY][M/G] 
-Xms[MEMORY][M/G] 
-jar [MINECRAFT SERVER JAR NAME].jar 
nogui
 
java -Xmx1024M -Xms1024M 
-jar minecraft_server.1.20.1.jar 
nogui
Full Generic Command 
[FOLDER DIRECTORY OF REQUIRED JAVA VERSION] java 
-Xmx[#]G -Xms[#]G 
[-XX:][SERVER OPTIONS] -jar 
[SERVER JAR NAME].jar 
nogui PAUSE
 
/usr/java/jdk1.8.0_351-aarch64/bin/java 
-Xmx8G -Xms8G -XX:+UnlockExperimentalVMOptions 
-XX:+UseG1GC -XX:G1NewSizePercent=20 
-XX:G1ReservePercent=20 -XX:MaxGCPauseMillis=50 
-XX:G1HeapRegionSize=32M -Dfml.readTimeout=180 
-jar forge-1.12.2-14.23.5.2847-universal.jar 
nogui PAUSE

IMPORTANT NOTE: If you end up using another port other than 25565, you must open the ports through the Primary VNIC and through your terminal. Also, in the "server.properties" file of your Minecraft server, be sure to change the port from 25565 to your new port.

NOTE: After running the server for the first time, it will stop itself. This is caused by the "eula.txt" file. This file is a text file that requires you to comply with the EULA terms in order to run the server. Simply set the value inside to "true" and save the file. (You might want to press the refresh button in WinSCP to see the update files). Rerun the server again.

NOTE: If you are setting up a modified Minecraft server to change the experience of Minecraft, simply substitute [SERVER JAR NAME] with the Forge server .jar file to run the server. Do NOT run the other server .jar file, otherwise it will create a Minecraft server without modifications.

ASIDE: Sometimes a modpack server folder may contain its own run command shell or bash script. However, I highly recommend creating your own run commands as going through multiple files to change server variables is very cumbersome.
