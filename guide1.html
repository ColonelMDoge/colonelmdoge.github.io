How To Setup A Cloud Oracle Server

Who Should Read This Guide?

If you are looking for a way to run a server absolutely for free while getting access to reliable computer power, then this guide is for you! It was constructed based on previous research to solve this exact problem. (Specifically, how would someone run a 24/7 Minecraft server that is not performatively underwhelming compared to online free services?)

What is an OCI?

OCI, or Oracle Cloud Infrastructure, is a cloud platform developed by the Oracle Corporation who also obtained the rights to the JAVA language by purchasing Sun Microsystems, the company who created the language. OCI allows anyone to create their own compute instances to manage databases, storage blocks, servers, and websites. The platform's "Always Free" plan allows a taste for the dozens of services that they provide, whether it be IaaS, SaaS, DaaS, or PaaS. The following below demonstrates how to setup a Cloud Compute Instance.

For more information, head to https://www.oracle.com/cloud/why-oci/ before continuing.

NOTE: A credit card is required to start the "Always Free" plan. You may be charged a miniscule amount to verify the card, but it will be refunded soon after.

Navigating Through The Cloud

Head to https://www.oracle.com/cloud/free/ to get started. During the sign up process, you will need to enter information such as your name, date of birth, home address, email, credit card information, etc. You will have to setup MFA as well. After successfully signing up, log in to your account to access the Oracle Cloud.

Under "Service Links", click on "Instances Compute". If it is not there, search for it using the search bar. Press the "Create Instance" button to create a new instance.

Choose a name for your instance. An example could be: "MyInstance".

Scroll down to "Image and shape". Click on the "Edit" button on the top right to take you to the details of the processor. Ignore the "Image" and go to the "Shape". Choose the "Ampere Arm-based processor" and check the "VM.Standard.A1.Flex". For the "Always Free" plan, you are limited to 4 OCPUs and 24GB RAM of all Ampere processors combined. If you do not have enough OCPUs or RAM for your next Ampere processor instance, simply choose the "Specialty and previous generation" processor. You will only get 1 OCPU and 1GB RAM, however, so partition your shapes wisely!

Continue on to the "Add SSH Keys" section and tick on "Generate a key pair for me". Download the private key and/or the public key. (The public key will not be used for later steps)

After everything is to your satisfaction, click on the "Create" button on the bottom. Congratulations, you have just made your very first instance!

Configuring Your Server's Network

Servers have default ports that are required to connect to the server. For example, Minecraft servers have the default port of 25565, and Mindustry servers have the default port of 6567. To allow these ports, you need to add them to a security list.

Go to your newly created instance and head to the "Primary VNIC" section. Click on the subnet link, and click on the default security list link. You will then be taken to the security list where you can add ingress rules.

Click on the "Add Ingress Rules" button. The source type should be "CIDR", and the source CIDR should be "0.0.0.0/0". Select which IP Protocol you require. Have the source port range selected to "All", and choose the destination port range (The default port of your server). The description is optional. Save the changes.

NOTE: You must add two ingress rules: one for the TCP IP Protocol, and one for the UDP IP Protocol.

Using Your SSH Private Key

A Secure Shell, or SSH, is a encrypted network protocol that allows two computers to communicate with each other securely over an unsecure network. An SSH is used to remotely access an OCI to execute commands, such as starting up servers, or downloading libraries from the internet. The SSH used to access the OCI will be an application called PuTTY. Head to the website https://www.chiark.greenend.org.uk/~sgtatham/putty to download PuTTY.

Once the installer has been downloaded and executed, there will be two applications that will be added: PuTTY itself, and PuTTYgen. Referring back to "Navigating Through The Cloud", a private key was downloaded. This key is required to authorize access to the instance. Without the key, access will never be granted. This is why it is important to not lose the key file, as you will not be able to access the file again after the creation of your instance.

The key itself cannot be used to access the instance through PuTTY. It needs its own private key in order to authorize access. To do this, open the PuTTYgen app. Click on the "Load" button to load an existing key. Make sure the file type is "All Files", and select the private key you downloaded from the OCI website. Click on "Save private key", and name the new PuTTY key whatever you want.

Creating and Configuring Your Putty Session

After creating the private PuTTY key, open the PuTTY app. This is where you will create the SSH to gain access to your instance. Before you can access it, you need to list the required information to connect to the instance. Fill the "Host Name" with the IP of your instance (located at your instance on the OCI website). Check the SSH connection type and check the "Only on clean exit" for when you close PuTTY.

On the left of the PuTTY application is the "Category" section. Locate to Connection -> SSH -> Auth. In that tab is where you have to load in the generated PuTTY key required to access the instance. After it is done, go back to the "Session" category and go to the "Saved Sessions" section. Click on the "Default Settings" and press the "Save" button on the right to save all of the information the next time you come back to PuTTY. This way you do not need to reenter all the information every time you open PuTTY.

Click on the "Open" button on the bottom (or press "Enter" on your keyboard) to open the session. The default login is "opc" unless otherwise stated.

NOTE: NEVER share your PuTTY key or OCI key to ANYONE YOU DO NOT TRUST!
